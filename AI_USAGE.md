While working on the Issue 1 for test assessment, I used AI agent by following the strategies below:

1. Humans must guide AI.
Give instructions, don't let AI "think". Strategic dicisions like suggested workflow, bug-fix approach, etc. must be specified in the input prompt. Even for suggested solutions generated by AI agent, review them manually to check which is the best approach.
AI can think, but often in the way that human doesn't expect.
Do research out of the code box by using Google, Stackoverflow, ChatGPT, etc. Collect necessary information and make decisions.
Human-guided details are especially useful when dealing with low-performing LLMs in AI agents.
For example, Raptor Mini (Preview) and Grok Code Fast were used for this assessment, chosen by GitHub Copilot's Auto selection.
2. Use clear prompt to expect better results. Add as many details as possible in input prompts, but keep concise. AI agents work better with high-quality input (concise and organized prompt structure), so using refined prompt is a good choice to improve agent's accuracy. This is especially useful when prompts are long.
3. Don't ellaborate intructions into multiple prompt where unecessary. AI agent's lose old chat history, considering it less important sometimes.
